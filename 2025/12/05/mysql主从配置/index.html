常见问题及处理

问题 可能原因 解决方法
Slave_IO_Running: Connecting 网络不通、用户密码错、防火墙 检查 ping、mysql -u repl -p -h 主库IP、开放3306端口
Slave_SQL_Running: No SQL语句执行错误（如主键冲突） 查看 Last_Error，跳过错误：SET GLOBAL SQL_SLAVE_SKIP_COUNTER=1; START SLAVE;
主从延迟大 从库性能差、大事务、网络慢 升级从库硬件、拆分大事务、开启并行复制
主库宕机后提升从库为主 需要手动切换 从库执行 STOP SLAVE; RESET SLAVE ALL; 然后修改应用指向新主库
