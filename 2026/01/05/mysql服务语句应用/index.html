一、SQL语言分类

分类 全称 作用 常用语句 关注人群
DDL Data Definition Language 管理库、表、索引、用户等基础结构 CREATE, ALTER, DROP 运维+开发
DCL Data Control Language 权限与安全控制 GRANT, REVOKE, COMMIT, ROLLBACK 运维
DML Data Manipulation Language 操作表中数据记录 INSERT, UPDATE, DELETE, SELECT 开发
DQL Data Query Language 查询数据记录 SELECT 运维+开发

💡 实际中 SELECT 常归入DML，但单独列为DQL更强调查询逻辑。

三、常用数据类型

3.1 整数类型

类型 字节 范围（有符号）
tinyint 1 -128 ~ 127
smallint 2 -32768 ~ 32767
int 4 -21亿 ~ 21亿
bigint 8 ±9.22×10¹⁸

3.2 浮点/定点类型

  • float(m,d) / double(m,d)：近似值，存在精度损失
  • decimal(m,d)：精确定点数（推荐金融场景）

3.3 字符串类型

类型 特点 最大长度
char(n) 固定长度，检索快 255字符
varchar(n) 可变长度，节省空间 65535字符
text 长文本，不能有默认值 65535字符

char vs varchar：char会用空格补位，取出时去掉；varchar额外+1字节存储长度。
varchar vs text：varchar可指定n，可直接建索引；text需指定前缀长度才能索引。

3.4 时间类型

  • date：YYYY-MM-DD
  • datetime：YYYY-MM-DD HH:MM:SS（8字节）
  • timestamp：自动更新为修改时间（4字节）

四、约束与属性

约束 说明
PRIMARY KEY 主键，非空且唯一，每表一个
UNIQUE KEY 唯一约束
NOT NULL 非空约束
FOREIGN KEY 外键（多表关联，较少用）
属性 说明
DEFAULT 默认值
AUTO_INCREMENT 自增（常配合主键）
COMMENT 注释
UNSIGNED 非负数值

模块 核心要点
