🚨 常见错误速查

现象 可能原因 排查命令 / 日志位置
Connection refused rsyncd 未启动 / 防火墙 systemctl status rsyncd；firewall-cmd --add-service=rsync
@ERROR: auth failed 用户名或密码错误 检查 /etc/rsyncd.secrets 格式和权限（600）
@ERROR: Unknown module 模块名写错或未定义 rsync 服务端IP:: 查看列表；检查 rsyncd.conf 中模块名
@ERROR: chdir failed 服务端 path 目录不存在 检查 path = /data/backup 目录是否创建且权限正确
Permission denied (13) 服务端目录权限不足 ls -ld /data/backup；确保 uid/gid 用户有写入权限
rsync: mkstemp failed 文件创建权限问题 服务端目录权限；SELinux 阻止（setenforce 0 测试）

5. 小贴士

  • 修改 rsyncd.conf 后执行 systemctl restart rsyncd 生效。
  • 密码文件权限必须为 600，否则 rsync 拒绝使用。
  • 生产环境建议结合 hosts allow 限制访问 IP。
  • 同步时注意末尾斜杠差异：
    /local/dir/ → 同步目录内容到目标
    /local/dir → 将目录本身同步到目标（产生子目录）
  • 若防火墙开启，需允许 TCP 873 端口：firewall-cmd --permanent --add-port=873/tcp && firewall-cmd --reload

Rsync 问题都能从 /var/log/rsyncd.log 和 rsync :: 测试中找到答案。**
