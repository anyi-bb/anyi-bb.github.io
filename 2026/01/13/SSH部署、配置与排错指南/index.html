🚨 常见错误速查

现象 可能原因 排查位置/命令
Connection refused sshd 未启动 / 端口不对 / 防火墙 systemctl status sshd；firewall-cmd --add-service=ssh
Permission denied (publickey,password) 密码错 / 禁用了密码登录 / 密钥未添加 /var/log/secure；检查 sshd_config 中的 PasswordAuthentication / PubkeyAuthentication
Host key verification failed 服务端重装系统，指纹变化 删除 ~/.ssh/known_hosts 中对应行
Connection closed by remote host hosts.deny 限制 / 达到 MaxAuthTries /var/log/secure 会记录拒绝原因
User xxx not allowed AllowUsers 未包含该用户 检查 sshd_config 中的 AllowUsers / DenyUsers
登录后立即断开 用户的 shell 配置错误（如 /bin/false） /var/log/secure 提示 failed to exec shell

5. 小贴士

  • 改 sshd_config 后务必 systemctl restart sshd 前多开一个会话，避免改错把自己锁在外面。
  • 生产环境建议：禁用 root 登录 + 禁用密码 + 使用密钥 + 改高位端口。
  • 防火墙开放端口：firewall-cmd --permanent --add-port=2222/tcp && firewall-cmd --reload
  • SELinux 可能阻止自定义端口：semanage port -a -t ssh_port_t -p tcp 2222

SSH 问题都能从 /var/log/secure 和 ssh -vvv 里找到答案。**
