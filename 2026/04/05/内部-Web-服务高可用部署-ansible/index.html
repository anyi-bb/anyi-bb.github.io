内部 Web 服务高可用部署+ansible

发表于2026-04-05|更新于2026-04-05
|浏览量:

roles对playbook的一种重新编排。
[root@m01 ~]# mkdir roles
[root@m01 ~]# cd roles/
[root@m01 roles]#
#通过命令生成roles角色目录
[root@m01 roles]# ansible-galaxy init backup

项目目录结构

1. ansible.cfg（用于关闭 host key 检查）

2. inventory.ini

3. group_vars/all.yml（全局变量）

4. site.yml（入口 Playbook）

5. 角色 common（系统初始化）

roles/common/tasks/main.yml

roles/common/handlers/main.yml

6. 角色 mysql

roles/mysql/tasks/main.yml

roles/mysql/handlers/main.yml

主要任务：

  • 安装 MySQL 5.7
  • 启动服务
  • 获取临时密码，修改 root 密码
  • 创建数据库、创建用户并授权
  • 修改 bind-address 允许远程连接

其中用到 mysql_db 和 mysql_user 模块，它们直接操作 MySQL。

7. 角色 web

roles/web/tasks/main.yml

roles/web/templates/nginx-default.conf.j2

roles/web/templates/index.php.j2

roles/web/handlers/main.yml

  • 安装 nginx, php-fpm, php-mysqlnd 等
  • 修改 php-fpm 配置文件（监听 9000 端口）
  • 通过 template 模块把 nginx-default.conf.j2 和 index.php.j2 渲染成真正的配置文件，放到目标机器上
  • 启动服务

8. 角色 lb（负载均衡 + Keepalived）

roles/lb/tasks/main.yml

roles/lb/templates/nginx-upstream.conf.j2

roles/lb/templates/keepalived.conf.j2

roles/lb/handlers/main.yml

  • 安装 nginx, keepalived
  • 通过模板生成 upstream.conf（包含后端 web 服务器列表）
  • 通过模板生成 keepalived.conf（根据 keepalived_state 和 keepalived_priority 生成主备不同的配置）
  • 启动服务

9. 执行部署

  1. 配置 SSH 免密登录（控制机到所有目标节点）：

  2. 测试连接：

  3. 运行 Playbook：

  4. 验证：

    • 浏览器访问 http://192.168.10.100/index.php，刷新几次看到主机名在 web01 和 web02 间轮询。
    • 停止主负载均衡器的 keepalived：systemctl stop keepalived，VIP 自动漂移至备机，服务不中断。
