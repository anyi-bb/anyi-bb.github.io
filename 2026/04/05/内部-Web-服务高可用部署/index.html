项目部署文档：企业内部 Web 服务高可用部署

1. 项目概述

为模拟中小型企业内部 Web 服务的生产环境，在虚拟机环境中搭建一套高可用、负载均衡的 LNMP 架构。实现：

  • 两台 Web 服务器提供相同内容
  • Nginx 反向代理实现负载均衡
  • Keepalived 提供 VIP 高可用，主备自动切换
  • Ansible 实现一键自动化部署

2. 硬件环境

设备类型 配置建议 数量 用途
物理机/笔记本 8GB+ 内存，50GB+ 空闲磁盘 1 运行虚拟机
虚拟机 2 核 CPU，2GB 内存，20GB 硬盘 4 模拟服务器节点（见下表）

2.1 虚拟机规划（使用 VMware Workstation 或 VirtualBox）

主机名 IP 地址 角色 操作系统
lb01 192.168.10.10 主负载均衡器 + Keepalived CentOS 7.9
lb02 192.168.10.11 备负载均衡器 + Keepalived CentOS 7.9
web01 192.168.10.20 后端 Web 服务器 1 CentOS 7.9
web02 192.168.10.21 后端 Web 服务器 2 CentOS 7.9
db01 192.168.10.30 MySQL 数据库服务器 CentOS 7.9

注：若资源有限，可将 db01 与任意一台 Web 合并，但为了清晰演示，建议独立。

VIP（虚拟 IP）：192.168.10.100（对外提供服务）

7. 故障排查记录（常见问题）

现象 可能原因 解决方法
访问 VIP 出现 502 Bad Gateway PHP-FPM 未启动或 Nginx 配置错误 检查 php-fpm 状态，查看 Nginx error log
Keepalived VIP 不切换 防火墙阻止 VRRP 协议 关闭防火墙或允许协议 112
后端 Web 节点无法连接 MySQL 数据库权限未正确设置 在 db01 上执行 GRANT 并 FLUSH PRIVILEGES
Ansible 连接失败 SSH 密钥未配置或 hosts 解析错误 用 ansible all -m ping 测试

8. 项目成果总结

  • 完成 4 节点高可用 LNMP 环境部署，故障切换时间 < 3 秒
  • 通过 Ansible 实现重复部署时间从 1 小时降至 5 分钟
